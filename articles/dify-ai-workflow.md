## 零基础实战：3步用Dify搭建你的第一个AI工作流

我们以一个最实用的场景为例：**搭建一个能自动总结网页文章内容的智能助手**。你只需要一个URL，它就能返回文章的核心摘要。

### 第1步：环境准备与创建应用

首先，你需要访问Dify的官网并注册一个账号。它提供了云端版本，直接使用非常方便。

1.  登录后，进入主控制台。
2.  点击“创建应用”，选择“**聊天助手**”类型（这是最基础、最容易上手的类型）。
3.  给你的应用起个名字，比如“文章总结大师”，然后点击创建。你就进入了一个可视化的应用编排界面。

### 第2步：编排核心AI工作流（拖拽与配置）

这是最核心的一步。我们需要设计一个简单的工作流：接收网址 -> 抓取网页内容 -> 发送给大模型进行总结。

1.  **添加开始节点**：这个界面默认有一个“开始”节点。在右侧的配置栏里，你需要定义一个“输入变量”，比如 `url`，类型为“文本”，并设为必填。这样，用户使用时就必须输入一个网址。
2.  **添加“网页抓取”工具节点**：
    *   在左侧的“工具”列表里，找到“网页抓取”工具，将它拖拽到画布上，与“开始”节点连接起来。
    *   配置这个节点。在“输入变量”中，将“网页URL”这个字段，关联上我们第一步创建的 `url` 变量。
3.  **添加“LLM”大模型节点**：
    *   再拖一个“LLM”节点到画布上，并与“网页抓取”节点连接。
    *   选择你想要使用的大模型，比如GPT-3.5-turbo（成本较低，速度快）。
    *   在“提示词”(Prompt)部分，这是灵魂所在。你可以这样写：
        > “你是一个专业的摘要助手。请根据以下网页内容，用简洁的中文总结出核心要点，不超过200字：\n\n{{#context#}}”
    *   这里的 `{{#context#}}` 是一个变量，你需要点击它，然后选择上一个“网页抓取”节点输出的“网页内容”。这样，抓取到的文本就会自动填入提示词中。
4.  **连接结束节点**：将LLM节点的输出，连接到最终的“结束”节点。在“结束”节点的配置中，设置“回答”变量的内容为LLM节点的输出。

至此，一个完整的**AI工作流**就编排完成了。整个过程就像画流程图，非常直观。

### 第3步：调试、发布与使用

1.  **调试**：点击界面上方的“运行”按钮，会弹出一个对话框。在输入框里粘贴一个网页链接（例如一篇新闻报道），点击运行。稍等几秒，右侧就会显示AI为你生成的摘要。如果结果不理想，可以随时返回去调整LLM节点的提示词。
2.  **发布**：调试满意后，点击“发布”按钮。你可以选择“运行一次”生成一个临时链接，或者“创建为应用”来生成一个可长期访问的独立Web应用。
3.  **分享**：把生成的应用链接分享给你的同事或朋友，他们打开网页就能直接使用你的“文章总结大师”了，完全无需知道背后的原理。

## Dify的优缺点：它真的适合你吗？

在决定是否投入时间学习Dify之前，我们客观地看一下它的优缺点。

| 特性 | 优点 (为什么选择它) | 缺点 (你需要知道的事) |
| :--- | :--- | :--- |
| **上手难度** | **真正的零代码/低代码**，可视化拖拽，对非技术人员极度友好。 | 对于完全不理解“变量”、“节点连接”等基础概念的人，仍有一个轻微的学习曲线。 |
| **灵活性与功能** | 支持复杂的分支、循环逻辑，可接入丰富外部工具（搜索、数据库、各类API），潜力巨大。 | 高级功能（如自定义工具）仍需一定的技术背景来理解和调试。 |
| **模型支持** | 支持主流云端模型（OpenAI, Claude等）和**本地私有模型**，兼顾便捷与数据安全。 | 本地模型部署本身仍是一个技术活，Dify只是提供了一个使用接口。 |
| **成本** | 开源免费，云端版有免费额度。构建应用成本远低于传统开发。 | 大模型API调用费用需要用户自行承担，工作流复杂时成本需自行核算。 |
| **数据隐私** | 支持本地私有化部署，数据完全自主可控，适合企业内网场景。 | 云端版数据会经过Dify服务器，对数据极度敏感的企业需评估合规性。 |

*   **快速原型验证**：产品经理验证AI功能想法，无需等待开发排期。
*   **个人效率工具**：打造个性化的信息搜集、处理、创作助手。
